# Новые возможности YouTube Music Downloader

## 🎯 Что нового?

### 1. Кастомный путь загрузки (downloadPath)
Теперь вы можете указать любой путь для сохранения музыки, включая сетевые диски SMB!

**Пример использования в config.json:**

#### Локальный диск:
```json
{
  "downloadPath": "D:\\MyMusicLibrary"
}
```

#### Сетевой диск (SMB):
```json
{
  "downloadPath": "\\\\192.168.1.100\\Media\\Music"
}
```

или

```json
{
  "downloadPath": "\\\\server\\share\\Music"
}
```

**Важно:** Если `downloadPath` пустой (`""`), будет использоваться `outputDir` (по умолчанию `"Music"`).

### 2. Отслеживание прогресса в реальном времени

Теперь во время работы создается файл **progress.txt**, который показывает:

- 📊 **Общий прогресс** - сколько артистов завершено из общего количества
- 🎵 **Текущий артист** - какой артист обрабатывается сейчас
- 💿 **Текущий альбом** - какой альбом скачивается
- 🎧 **Текущий трек** - название текущего трека
- ⏱️ **Примерное время** - расчетное время до завершения всех загрузок
- ✅ **Статус завершения** - сообщение о завершении всех задач

**Пример содержимого progress.txt:**

```

📊 ОБЩИЙ ПРОГРЕСС:
   Артистов завершено: 1 из 3 (33%)
   Прогресс: [████████████████                                  ] 33%

🎵 ТЕКУЩИЙ АРТИСТ: Linkin Park

╔═══════════════════════════════════════════════════════════════════╗
```

## 🚀 Как использовать?

1. **Укажите путь загрузки** в config.json (опционально):
   ```json
   {
     "downloadPath": "\\\\192.168.1.100\\Music"
   }
   ```

2. **Запустите скрипт:**
   ```bash
   npm start
   ```

3. **Откройте progress.txt** в любом текстовом редакторе для отслеживания прогресса в реальном времени

4. **Дождитесь завершения** - когда все загрузки завершатся, в progress.txt появится:
   ```
   ✅ ВСЕ ЗАГРУЗКИ ЗАВЕРШЕНЫ!
   ```

## 📝 Примеры конфигурации

### Базовая конфигурация (локальная папка):
```json
{
  "forceRedownload": false,
  "artists": ["FEDYA BOOKER", "TMNV", "FACE"],
  "outputDir": "Music",
  "downloadPath": ""
}
```

### Конфигурация с сетевым диском:
```json
{
  "forceRedownload": false,
  "artists": ["FEDYA BOOKER", "TMNV", "FACE"],
  "outputDir": "Music",
  "downloadPath": "\\\\nas\\media\\Music"
}
```

### Конфигурация с кастомным локальным диском:
```json
{
  "forceRedownload": false,
  "artists": ["FEDYA BOOKER", "TMNV", "FACE"],
  "outputDir": "Music",
  "downloadPath": "E:\\MusicCollection"
}
```

## 💡 Советы

- **progress.txt обновляется автоматически** - просто откройте файл в редакторе с автообновлением (например, VS Code)
- **SMB пути работают только если сеть доступна** - убедитесь, что у вас есть доступ к сетевому диску
- **Путь в downloadPath имеет приоритет** - если он указан, outputDir игнорируется
- **Файл progress.txt создается автоматически** - не нужно создавать его вручную

## ⚠️ Важные замечания

1. При использовании SMB путей:
   - Убедитесь, что сетевой диск доступен
   - Проверьте права доступа на запись
   - Используйте правильный формат: `\\server\share\path`

2. Файл progress.txt:
   - Создается при запуске скрипта
   - Обновляется в реальном времени
   - Сохраняется после завершения работы

3. Время завершения:
   - Рассчитывается на основе средней скорости обработки артистов
   - Может меняться в зависимости от размера альбомов и скорости интернета
   - Становится более точным после обработки нескольких артистов
